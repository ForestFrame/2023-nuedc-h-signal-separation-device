# 2023 电赛 H 题 信号分离装置

2023 年全国大学生电子设计竞赛 H 题项目，项目目标是对混合信号进行识别与分离，并恢复输出两路目标信号。整套系统采用 `STM32 + FPGA` 的协同方案：

- `STM32` 负责混合信号采样、FFT 频谱分析、频率识别、波形分类和参数下发
- `FPGA` 负责串口参数接收、频率字/相位字换算以及双路波形重建输出

项目最终获省一等奖。

## 项目介绍

比赛题目要求将两路输入信号 `A`、`B` 经过叠加形成混合信号 `C = A + B`，然后从 `C` 中识别出两路原始信号的频率、波形类型等信息，并输出两路恢复后的信号 `A'`、`B'`。

该工程采用的是“先识别、后重建”的方案：

1. 前级模拟电路完成两路信号混合与调理
2. STM32 对混合信号进行定时采样和频谱分析
3. 根据频谱结果判断两路信号的频率和波形类别
4. 将识别结果通过串口发送给 FPGA
5. FPGA 根据参数完成两路信号的 DDS 重建并输出

## 项目架构

系统整体采用“先识别、后重建”的协同架构。比赛阶段绘制的系统框图如下：

![系统框图](doc/%E7%94%B5%E8%B5%9B%E6%A1%86%E5%9B%BE.jpg)

结合这张框图，系统可以分为前端信号链、STM32 识别链和 FPGA 重建链三部分：

- 前端信号链：
  - 两路输入信号 `A`、`B` 先经过模拟加法与前端调理，形成混合信号 `C`
  - STM32 实际处理的对象不是原始两路信号，而是混合后的 `C = A + B`
- `STM32` 侧识别链：
  - 使用 `TIM3` 产生稳定采样时序
  - 使用 `ADC1` 对混合信号进行采样
  - 使用 `DMA` 连续搬运一批采样点到内存缓冲区
  - 对采样结果做电压换算与归一化处理
  - 执行 `512` 点 FFT，得到幅值谱
  - 对频谱主峰和次峰进行搜索，完成频率识别
  - 根据频谱特征和阈值完成波形分类
  - 将识别得到的波形类型、频率、相位和补偿参数通过串口发送给 FPGA
  - 同时支持 HMI/串口参数调试，便于比赛现场快速校准
- `FPGA` 侧重建链：
  - 接收 STM32 下发的串口参数帧
  - 对 A/B 两路参数进行解包
  - 解析波形类型、频率参数和相位参数
  - 将频率参数换算为 DDS 所需频率字
  - 将相位参数换算为相位控制字
  - 根据波形选择信号完成查表输出
  - 最终生成双路恢复信号并驱动 DAC 输出

从职责划分上看，这套方案的核心思想是：

- `STM32` 负责“测量、分析、识别、控制”
- `FPGA` 负责“实时重建、并行输出”

这种拆分方式让 MCU 发挥算法与控制优势，让 FPGA 发挥高速并行和稳定输出优势，也使整个系统的调试边界更加清晰。

## STM32 侧工程实现设计

STM32 侧是整个系统的“识别核心”，主要任务是把混合信号转换成可供 FPGA 重建的参数信息。

### 采样率设计

这一部分是 STM32 侧实现中非常关键的设计点。

本项目采用 `512` 点 FFT，为了让题目中 `20kHz ~ 100kHz` 的候选频率尽量落在 FFT 的整数频点上，采样率经过了专门设计。工程中最终使用的采样频率为：

```text
Fs = 1.28 MHz
```

这样可得到频率分辨率：

```text
Δf = Fs / N = 1.28 MHz / 512 = 2.5 kHz
```

对应到比赛频率范围时：

- `20 kHz -> 第 8 个频点`
- `25 kHz -> 第 10 个频点`
- `30 kHz -> 第 12 个频点`
- `...`
- `100 kHz -> 第 40 个频点`

从采样窗口角度看，在 `512` 点、`1.28MHz` 采样率下，总采样时长为：

```text
T = N / Fs = 512 / 1.28 MHz = 400 us
```

这样一来，比赛中的候选频率在单个采样窗口内对应的周期数也刚好是整数：

- `20 kHz -> 8` 个周期
- `25 kHz -> 10` 个周期
- `30 kHz -> 12` 个周期
- `100 kHz -> 40` 个周期

这实际上就是一种相干采样设计思路。由于信号在采样窗口内包含整数个周期，FFT 时频率成分能够更集中地落在目标频点上，从而有效减小频谱泄露，降低相邻频点能量扩散对主峰识别的影响。

这样做的意义在于：

- 让候选频率尽可能对齐 FFT 整数 bin
- 让采样窗口尽可能满足相干采样条件
- 降低频谱泄露对主峰识别的影响
- 便于后续直接通过频点索引换算频率

从工程实现上，采样率由 `TIM3` 触发 `ADC1` 完成，定时器参数经过精确配置后得到目标采样频率。

### TIMER + ADC + DMA 设计

STM32 侧采样链路采用了 `TIM3 + ADC1 + DMA` 的组合方案。

设计思路如下：

- 使用 `TIM3` 产生固定频率的触发事件
- 使用 `ADC1` 在外部触发下进行采样
- 使用 `DMA` 自动将采样结果搬运到内存缓冲区
- 缓冲区采满后进入回调函数做后续处理

这样设计有几个明显好处：

- 采样周期由硬件定时器保证，稳定性高
- DMA 负责搬运数据，CPU 不需要逐点参与
- 采满一批数据后再统一处理，适合 FFT 运算

本项目中各外设的关键配置参数如下：

- `TIM3`：
  - `Prescaler = 9`
  - `Period = 4`
  - `CounterMode = TIM_COUNTERMODE_DOWN`
  - `ClockDivision = TIM_CLOCKDIVISION_DIV1`
  - `MasterOutputTrigger = TIM_TRGO_UPDATE`
  - 作用：产生固定频率的更新事件，作为 ADC 外部触发源
- `ADC1`：
  - `ClockPrescaler = ADC_CLOCK_SYNC_PCLK_DIV2`
  - `Resolution = ADC_RESOLUTION_6B`
  - `ScanConvMode = DISABLE`
  - `ContinuousConvMode = DISABLE`
  - `ExternalTrigConvEdge = ADC_EXTERNALTRIGCONVEDGE_RISING`
  - `ExternalTrigConv = ADC_EXTERNALTRIGCONV_T3_TRGO`
  - `DataAlign = ADC_DATAALIGN_RIGHT`
  - `NbrOfConversion = 1`
  - `DMAContinuousRequests = ENABLE`
  - `EOCSelection = ADC_EOC_SINGLE_CONV`
  - 采样通道：`ADC_CHANNEL_3`
  - 采样时间：`ADC_SAMPLETIME_3CYCLES`
  - 作用：在 `TIM3` 更新事件触发下，对单通道混合信号进行定时采样
- `DMA`：
  - `Instance = DMA2_Stream0`
  - `Channel = DMA_CHANNEL_0`
  - `Direction = DMA_PERIPH_TO_MEMORY`
  - `PeriphInc = DMA_PINC_DISABLE`
  - `MemInc = DMA_MINC_ENABLE`
  - `PeriphDataAlignment = DMA_PDATAALIGN_WORD`
  - `MemDataAlignment = DMA_MDATAALIGN_WORD`
  - `Mode = DMA_NORMAL`
  - `Priority = DMA_PRIORITY_LOW`
  - `FIFOMode = DMA_FIFOMODE_DISABLE`
  - 作用：将 ADC 采样结果按批量方式搬运到内存缓冲区，采满 `512` 点后触发后续处理

对应的软件处理流程为：

1. 启动串口中断接收
2. 启动 ADC 的 DMA 采样
3. 启动定时器
4. DMA 采满 512 点后触发回调
5. 在回调中完成电压换算、FFT 和波形识别

### FFT 频谱分析设计

FFT 用于把时域采样信号转换到频域，从而提取混合信号中的主要频率成分。

本项目中 FFT 的处理流程如下：

1. 将 ADC 采样结果换算为浮点电压数组

   - 输入：`dmaDataBuffer[]` 中的 ADC 原始采样值
   - 处理：按设定的电压换算公式转换为浮点电压序列
   - 输出：时域采样数组 `Volt[]`
   - 作用：把 ADC 数值转换为后续 DSP 计算更方便处理的浮点信号
2. 将实数序列组织为复数输入格式，虚部补零

   - 输入：时域采样数组 `Volt[]`
   - 处理：按照 CMSIS-DSP 复数 FFT 的输入格式，把实部写入偶数位，虚部统一补 `0`
   - 输出：复数输入数组 `processArr[]`
   - 作用：将原本的实数时域信号组织成 FFT 可直接处理的复数序列
3. 调用 CMSIS-DSP 库的 `arm_cfft_f32`

   - 输入：复数输入数组 `processArr[]`
   - 处理：执行 `512` 点复数 FFT
   - 输出：频域复数谱，结果仍保存在 `processArr[]` 中
   - 作用：将时域信号转换为频域信号，使主要频率成分和谐波分量能够在频谱中体现出来
4. 对 FFT 输出结果取复数模长

   - 输入：FFT 得到的复数频谱 `processArr[]`
   - 处理：调用 `arm_cmplx_mag_f32` 计算每个频点的模值
   - 输出：幅值数组 `testOutput[]`
   - 作用：把复数频谱转换为更直观的幅值谱，便于做峰值搜索和频率判定
5. 进行单边谱归一化，得到幅值谱数组 `amp[]`

   - 输入：幅值数组 `testOutput[]`
   - 处理：对直流分量和其余频点分别做归一化处理，得到单边谱幅值
   - 输出：最终用于识别的幅值谱数组 `amp[]`
   - 作用：将不同频点的幅值统一到可比较的尺度，便于后续进行阈值判断和波形分类

整体来看，这一处理链路中的数据形态是逐步变化的：

- `dmaDataBuffer[]`：ADC 原始采样值
- `Volt[]`：浮点时域电压信号
- `processArr[]`：FFT 输入/输出使用的复数数组
- `testOutput[]`：频谱模值结果
- `amp[]`：归一化后的单边幅值谱

由于本项目的采样率和 FFT 点数是配套设计的，所以后续可以直接通过频点索引乘以 `2.5kHz` 得到候选频率，大大简化了频率识别逻辑。

### 波形识别设计

在获得幅值谱之后，STM32 侧需要完成两件事：

- 判断两路信号的频率
- 判断两路信号的波形类型

本项目采用的是较为直接的工程化方法：

- 先在频谱中寻找最大的两个峰值
- 通过两个主峰所在频点判断两路频率
- 根据峰值分布和设定阈值区分正弦波与三角波

这一方法的基础在于：

- 正弦波的能量主要集中在基波
- 三角波除了基波外还会表现出明显的奇次谐波特征

因此，在固定采样链路和幅值范围下，可以通过经验阈值对不同波形进行稳定区分。为了方便现场调试，阈值还支持通过串口动态调整。

## FPGA 侧工程实现设计

FPGA 侧负责根据 STM32 下发的识别结果完成双路信号重建。

### 串口接收设计

STM32 完成混合信号识别后，不会直接把波形采样值传给 FPGA，而是把“描述波形”的参数通过串口发送给 FPGA。也就是说，STM32 与 FPGA 之间传输的是参数协议，而不是原始波形数据。

在 STM32 侧，每次会分别向 FPGA 发送 A 路和 B 路两帧数据。每帧的结构可概括为：

```text
[HEAD] [A/B 标识] [波形类型] [频率] [误差符号] [误差值] [相位参数] [END]
```

其中：

- `HEAD = 0x55`
- `A/B 标识 = 0x0A / 0x0B`
- `波形类型` 用于区分正弦波、三角波等
- `频率` 表示识别得到的标称频率
- `误差符号 + 误差值` 用于做更细粒度的频率补偿
- `相位参数` 用于后级相位控制
- `END = 0xAA`

这样设计的好处是串口协议简单、数据量小，而且能把 STM32 侧的识别结果完整传递给 FPGA 的重建链路。

FPGA 侧串口模块负责：

- 接收来自 STM32 的参数帧
- 区分 A 路与 B 路数据
- 提取波形类型、频率参数和相位参数
- 将接收结果缓存到寄存器中

从功能上看，这一层起到的是“参数解包”和“重建输入准备”的作用。它把串口协议转换成后续 DDS 模块可直接使用的数据格式，为双路波形发生器提供输入。

在 Verilog 实现上，这一部分的交互流程如下：

1. `uart_rx` 按字节接收串口数据
2. 当检测到帧头 `0x55` 时，开始进入收包状态
3. 中间各字节按顺序写入临时寄存器 `DATA_store`
4. 当检测到帧尾 `0xAA` 时，认为一帧接收完成，并拉高 `updata_flag`
5. 根据帧中的 A/B 标识，把当前帧写入 `A_reg` 或 `B_reg`

因此，`A_reg` 和 `B_reg` 可以理解为 FPGA 侧对两路波形控制参数的缓存中心。后续所有频率、相位和波形选择控制，都是从这两个寄存器中拆分得到的。

### 参数拆分与控制生成

`stm32_control.v` 位于串口接收模块之后，作用并不是直接收串口，而是对 `A_reg`、`B_reg` 中已经缓存好的参数进行拆分和整理。

这一层主要完成两类工作：

- 直接提取字段：
  - 例如从 `A_reg` / `B_reg` 中直接取出波形类型 `f_type_A`、`f_type_B`
  - 直接取出相位参数 `pha_x_A`、`pha_y_A`、`pha_x_B`、`pha_y_B`
- 细化频率表示：
  - 将 STM32 发送过来的“标称频率 + 误差符号 + 误差值”进一步整理成更适合后级计算的形式
  - 最终输出 `fre_y_A`、`fre_z_A`、`fre_w_A` 以及 B 路对应参数

从设计意图上看，这一步是在把 STM32 发来的“粗粒度识别结果”转换为 FPGA 后级更方便使用的“细粒度控制参数”。这样做的目的是把频率补偿和借位/补位处理放在 FPGA 侧完成，便于直接送入 DDS 控制链路。

### DDS 信号发生器设计

这一部分参考 GitHub 项目 [fpga-dds](https://github.com/ForestFrame/fpga-dds)，这里不做详细展开。

在本项目中，DDS 部分主要承担以下职责：

- 将频率参数换算为频率控制字
- 将相位参数换算为相位控制字
- 根据波形选择信号决定查表输出的波形类型
- 输出两路重建后的波形数据

在本项目中，FPGA 后级的数据流可以概括为：

1. `stm32_control.v` 输出波形类型、频率参数和相位参数
2. `fre_pha_data_ctrl.v` 将频率参数换算为 DDS 所需的频率字 `fre_step`
3. `fre_pha_data_ctrl.v` 将相位参数换算为相位字 `pha_step`
4. `wave_ctrl.v` 根据 `wave_sel`、`fre_step` 和 `pha_step` 执行 DDS 查表输出
5. 最终形成两路 DAC 输入数据，实现双路重建波形输出

因此，STM32 与 FPGA 的协作关系可以总结为：

- `STM32` 负责“识别参数”
- `FPGA` 负责“根据参数实时重建波形”

整体上，FPGA 侧保证了重建过程的实时性和稳定性，是整个系统“输出恢复”能力的核心。

## 工程缺陷

从比赛结果上看，本项目最终满足了题目要求并通过了测评，但如果从工程实现角度复盘，整套系统仍然存在比较明显的局限性。

### 1. 系统是开环重建，而不是闭环锁定

比赛要求 `A'` 与 `A`、`B'` 与 `B` 在示波器上能够连续、稳定、同频显示。当前工程采用的是“识别参数后直接重建”的开环方案：

- STM32 对混合信号进行一次采样和分析
- 提取波形类型、频率、相位等参数
- FPGA 根据这些参数用 DDS 方式重新生成信号

在这个过程中，原始模拟信号本身并没有被持续跟踪，也没有反馈链路去实时修正 FPGA 的输出。因此，从原理上讲，这套方案只能做到“按识别结果近似重建”，而不能做到像锁相环那样始终跟随原始信号。

### 2. 非同源时钟导致长期漂移不可避免

本项目中，FPGA 的 DDS 输出时钟与外部信号发生器的时钟并不是同一个时钟源。只要两端不是同源时钟，就一定会存在微小频率误差。即使这个误差非常小，随着时间积累也会逐渐体现为相位偏移。

这意味着：

- DDS 重建信号在短时间内可以做到频率非常接近
- 但在长时间观察下，仍然会相对原始信号发生缓慢漂移
- 在示波器上表现为波形无法永久静止重合，而是会慢慢移动

因此，严格来说，在没有同源参考时钟或闭环锁相机制的情况下，不可能真正实现无限时长的连续稳定同频显示。

### 3. 当前方案更依赖题目边界条件

本项目能够取得较好效果，和比赛题目的约束条件密切相关，例如：

- 候选频率范围固定在 `20kHz ~ 100kHz`
- 频率点是离散可枚举的
- 输入波形类型有限
- 幅值范围和测试条件相对可控

这使得 STM32 侧可以使用固定采样率、固定 FFT 点数和经验阈值来完成识别。换句话说，这套方案在比赛场景下是高效可行的，但如果直接推广到更开放的工程场景，鲁棒性还不够强。

### 4. 波形识别存在经验参数依赖

当前的波形识别方法主要依赖频谱主峰、次峰以及经验阈值进行判断。这种方法实现简单、调试方便，适合比赛阶段快速落地，但也存在局限：

- 阈值依赖具体硬件链路和测试环境
- 前端幅值变化、噪声变化时需要重新调整参数
- 对更复杂波形或频率更接近的组合情况，识别稳定性会下降

因此，这部分更偏向“面向赛题优化”的工程实现，而不是通用意义上的高鲁棒性识别算法。

### 5. 结果上满足要求，但原理上仍有边界

虽然这套系统最终通过了比赛测评，而且效果达到了预期，但它的本质更接近“在误差允许范围内完成高精度参数化重建”，而不是“严格意义上的原始信号同步复制”。

比赛现场允许的最大误差大约为 `20Hz`，而本项目实际重建误差控制在 `5Hz` 以内，已经明显优于题目要求。因此，在比赛评价标准下，该方案是成功的；但如果以更严格的工程标准去衡量，系统仍然受到开环结构和非同源时钟的根本限制。

## 使用方法

### STM32 工程

- 顶层设计位于 `FPGA/rtl/competition.v`
- 编译后下载到目标 FPGA
